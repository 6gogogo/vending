2	获取提货码
2.1	获取提货码
tapi/trade/code
出货指令

URL
http://$AUTHSERVER/osapi/trade/code

支持格式
JSON

HTTP请求方式
POST

请求参数
 无

注意事项
无。






请求消息内容
 请求参数	必选	类型及范围	说明
clientId	true	string	身份标识，clientId为平台为用户分配唯一身份标识
product_code	true	string	商品编码
out_of_time	false	long	提货码失效时间
sign	true	string	签名，请见签名方法
nonce_str	true	string	随机字符串，长度不长16位
notify_url	false	string	接收平台异步通知回调地址。此地址需为POST请求。将会以这种回调的方法把提货码的使用情况给予通知，详细参见回调通知



返回结果
 请求参数	必选	类型及范围	说明
return_code	true	string	SUCCESS/FAIL
return_msg	false	string	当return_code为FAIL时有此字段，说明错误原因
当return_code为SUCCESS时包含以下字段
result_code	true	string	SUCCESS
当return_code与result_code都为SUCCESS时包含以下字段
code	true	string	提货码
out_of_time	false	string	失效时间,如果请求参数中timeout存在，此参数会存在



回调通知
接口中提交上来的notify_url，平台得到用户在设备上的使用提货码结果后通过notify_url使用结果给予通知。
如果平台在通知时，发现用户返回的不是成功或超时，平台认为通知失败，接下来平台会通过一定的策略来定期发送通知。
 通知字段	必选	类型及范围	说明
return_code	true	string	SUCCESS/FAIL
			
当return_code为SUCCESS是包含以下字段
code	true	string	提货码
result_code	true	string	SUCCESS/FAIL
cid	true	string	货柜编号，如果出货为主柜cid=assetId，否则为货柜编号
locationId	true	string	货柜的货道编号
sign	true	string	签名，请见签名方法
nonce_str	true	string	随机字符串，长度不长16位
orderNo	true	string	订单编号
deliverStatus	true	string	出货状态，状态如下： 0 出货成功 、 1 售空，无法出货 其它状态后面补上
exec_ts	true	long	出货执行完时间，格式为 1970-01-01至今的秒数
createTime	true	long	订单的创建时间
