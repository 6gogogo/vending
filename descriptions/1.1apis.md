# 接口说明
## 接口说明
测试平台地址：http://pre.smartvm.cn

## 请求方式
ContentType：application/json

Method：POST

## 公共请求参数
| 参数 | 类型 | 必选 | 说明 |
| --- | --- | --- | --- |
| clientId | String | 是 | 系统分配的唯一ID |
| nonceStr | String | 是 | 随机字付串 |
| sign | String | 是 | 签名信息 |


## 签名说明
1、对所有传入参数按照字段名的ASCII码**从小到大排序（字典序）**后，使用URL键值对的格式（即key1=value1&key2=value2…）拼接成字符串string1，**注意：值为空的参数不参与签名。**

2、在string1最后拼接上key得到stringSignTemp字符串，并对stringSignTemp进行md5运算，得到sign值signValue。<font style="color:#DF2A3F;">MD5值所有字母转换为大写</font>

3、如果请求参数中参数为一个对象，将对象内容以JSON字符串的形式拼接到stringSignTemp

```json
id=566fbf1244f4d4f90ab7d717&orderNo=1450163934464-709314&tradeNo=1005810748126
&locationId=1&assetId=91000092&cid=91000092&goodsId=563ab6570cf2e3ca2556f44a
&goodsName=统一冰绿茶&key=xxxxxxxxx
```

## 正确返回示例
```json
{
  "code":200,
  "message":"具体错误内容",
  "data":{
    //具体返回数据
  }
}
```

## 错误返回示例
```json
{
  "code":400,
  "message":"签名错误"
}
```

# 开门接口
## API名称
api/pay/container/opendoor

实现方：映翰通

## 请求参数
| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| userId | String | 是 | 用户的唯一ID |
| eventId | String | 是 | 本次开门的唯一事件ID |
| deviceCode | String | 是 | 设备编号 |
| payStyle | String | 是 | 支付方式<br/>2：微信  3：支付宝 |
| doorNum | String | 否 | 门编码<br/>1：右  2: 左   双门柜必传，单门柜可不传或传编号 1 |
| phone | String | 是 | 手机号 |


## 请求示例
```json
{
  "userId":"00000001",
  "eventId":"000000000000000000",
  "deviceCode":"9100003",
  "payStyle":"2",
  "doorNum":"1",
  "phone":"1861198283",
  "clientId":"xxxxxxxxxx",
  "nonceStr":"xxxxxxxxxx",
  "sign":"xxxxxxxxxxxxxxxxxxxxx"
}
```

## 返回示例
```json
{
  "code":200,
  "message":"请求成功",
  "data":{
    "orderNo":"91828323I3232323I23232332323"
  }
}
```

## 返回参数
| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| orderNo | String | 是 | 开门业务订单号，开门成功以后会使用此订单号生成订单 |


# 门状态推送
## API名称
实现方：第三方系统

## 请求参数
| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| eventId | String | 是 | 执行开门时事件id |
| deviceCode | String  | 是 | 设备编号 |
| status | String | 是 | OPENDING :  正在开门<br/><font style="color:#000000;background-color:#ffffff;">SUCCESS  :    开门成功</font><br/><font style="color:#000000;background-color:#ffffff;">CLOSED   :     门已关</font><br/><font style="color:#000000;background-color:#ffffff;">FAIL ：开门失败</font> |
| doorIsOpen | String | 否 | 关门时推送门是否拉开<br/>当status为CLOSED时有此字段<br/>Y：门拉开过<br/>N：未拉开门 |


## 请求示例
```json
{
  "eventId":"000000000000",
  "deviceCode":"90000001",
  "status":"SUCCESS",
  "clientId":"xxxxxxxxxx",
  "nonceStr":"xxxxxxxxxx",
  "sign":"xxxxxxxxxxxxxxxxxxxxx"
}
```

## 返回示例
```json
{
  "code":200,
  "message":"请求成功",
  "result":{
    "eventId":"000000000000000000"
  }
}
```

## 返回字段
| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| eventId | String | 是 | 处理成功以后，返回对应的事件id |
